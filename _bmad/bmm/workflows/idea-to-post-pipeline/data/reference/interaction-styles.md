# Interaction Styles: Когда Collaborative vs Autonomous

## 4-Mode Interaction Matrix

### CREATE MODE: Collaborative (50/50)

**Когда ты (user) контролируешь:**
- Выбор идей для исследования
- Выбор углов для написания
- Финальное одобрение постов

**Когда я (assistant) автономен:**
- Проведение исследования
- Генерация вариантов текста
- Предложение улучшений

**Процесс:**
```
User: "Создай пост про ИИ контент"
       ↓
Assistant: "Вот 5 углов для исследования, выбери 2-3"
       ↓
User: "Нравится угол 1 и 3"
       ↓
Assistant: "Вот 3 варианта текста с этими углами"
       ↓
User: "Вариант 1 хороший но нужно усилить hook"
       ↓
Assistant: "Вот улучшенная версия"
       ↓
User: "Идеально! Добавь в базу"
```

**SLA:** 2-3 часа на цикл (идея → 3 поста + метрики)

**Когда использовать:**
- Ты хочешь быть в процессе создания
- Хочешь учиться как пишется контент
- Есть время на взаимодействие
- Нужно убедиться что качество высокий

---

### EDIT MODE: Autonomous with Recommendations (70/30)

**Когда ты (user) контролируешь:**
- Финальное одобрение улучшений
- Выбор какие посты редактировать
- Выбор какие рекомендации применить

**Когда я (assistant) автономен:**
- Анализ какие посты нуждаются в улучшении
- Предложение конкретных улучшений
- Применение изменений (если ты одобришь)

**Процесс:**
```
Assistant: "Проверил все посты. 5 из них нуждаются в улучшении. Вот рекомендации:"
       ↓
User: "Окей, применяй улучшения для постов 1,2,3. Пост 4 и 5 оставь как есть"
       ↓
Assistant: "Готово! Вот улучшенные версии постов 1,2,3"
       ↓
User: "Отлично! Добавь в базу"
```

**SLA:** 30-60 мин на цикл (анализ 20 постов + улучшение 5 → публикация)

**Когда использовать:**
- Ты уже создал контент, теперь нужно оптимизировать
- У тебя нет времени на детальный процесс
- Ты хочешь рекомендации но сохранить контроль
- Нужно быстро улучшить низкорейтинговые посты

---

### VALIDATE MODE: Highly Autonomous (90/10)

**Когда ты (user) контролируешь:**
- Финальное решение что опубликовать
- Выбор какие посты нуждаются в доработке

**Когда я (assistant) автономен:**
- Проверка всех постов на 5 критериев
- Составление отчёта
- Выявление проблем

**Процесс:**
```
User: "Проверь все 9 постов перед публикацией"
       ↓
Assistant: (анализирует все посты)
        → Результат: 8 готовых, 1 нужна доработка
       ↓
User: "Отлично! Публикуй 8 готовых. Пост #6 переделай"
```

**SLA:** 10-30 мин на цикл (проверка 20 постов + отчёт)

**Когда использовать:**
- Нужна гарантия качества перед публикацией
- Хочешь быстро проверить большой объём контента
- Нужен отчёт о проблемах
- Не хочешь вдаваться в детали

---

### YOLO MODE: Fully Autonomous (100% AI, 0% user input)

**Когда ты (user) контролируешь:**
- ТОЛЬКО финальное решение: принять все / выбрать / или создать ещё

**Когда я (assistant) автономен:**
- ВСЕ шаги: идеи → исследование → написание → валидация → улучшение → варианты

**Процесс:**
```
User: "YOLO: 3 идеи → 9 постов с auto-fix"
       ↓
Assistant: (БЕЗ ПЕРЕРЫВОВ 2-3 минуты)
  → Добавляет идеи
  → Исследует параллельно
  → Пишет посты параллельно
  → Валидирует (5 проверок)
  → Auto-улучшает если нужно
  → Генерирует варианты
  → Показывает финальный отчёт
       ↓
User: "Отлично! Применяй всё" или "Выбери пост #1,3,5" или "Создай ещё"
```

**SLA:** 3-5 минут для 3 идей → 9 постов (вместо 6-8 часов)

**Когда использовать:**
- Ты очень занят и нужен контент ДЛЯ ПРЯМО СЕЙЧАС
- Хочешь быстро протестировать идеи
- Нужна демонстрация возможностей
- Content sprint/quick prototype

## Mode Selection Guide

```
Выбери режим в зависимости от твоей ситуации:

CREATE MODE (50/50):
  ✅ Если: Хочешь учиться, есть время, нужно высокое качество
  ❌ Если: Торопишься, много идей, нужен большой объём

EDIT MODE (70/30):
  ✅ Если: Уже есть контент, нужна оптимизация, нет много времени
  ❌ Если: Контент создаётся впервые, нужна помощь в процессе

VALIDATE MODE (90/10):
  ✅ Если: Много контента, нужна быстрая проверка, нужен отчёт
  ❌ Если: Контент только создаётся, нужна помощь в процессе

YOLO MODE (100% AI):
  ✅ Если: Очень срочно, нужно много контента, sprint/prototype
  ❌ Если: Нужна помощь на каждом этапе, хочешь учиться
```

## Interaction Best Practices

### CREATE MODE
- **Дай чёткие критерии:** "Напиши пост про ИИ контент для фаундеров"
- **Выбирай лучшие варианты:** Я предложу 3, ты выбираешь 1
- **Давай feedback:** "Усилить hook, добавить социальный proof"
- **Итеративно улучшай:** 2-3 раунда feedback это нормально

### EDIT MODE
- **Перепроверь рекомендации:** Не слепо применяй все предложения
- **Выбери что нужно:** "Применяй улучшения только для постов 1,2,3"
- **Добавляй контекст:** "Эта аудитория любит emotional triggers"
- **Одобри перед публикацией:** Всегда проверь изменения

### VALIDATE MODE
- **Используй отчёт:** Читай почему пост не прошёл валидацию
- **Применяй recommendations:** Я покажу что именно нужно улучшить
- **Выбери quality threshold:** "Публикуй только posts с score ≥85%"
- **Сохраняй отчёты:** Помогает видеть прогресс

### YOLO MODE
- **Будь ясен в spec:** "YOLO: 5 идей про ИИ + research + write + auto-fix"
- **Смотри отчёт:** Прежде чем принять результаты посмотри metrics
- **Выбирай оптимальные варианты:** "Применяй лучшие 3 варианта на каждый пост"
- **Переспрашивай если непонятно:** "Почему пост #2 отмечен как needs review?"
