# Step 2: Brownfield Documenter Modification Execution

## STEP GOAL:

Execute targeted modifications to the selected edit target using mode-appropriate editing methods.

## TARGETED MODIFICATION: Edit Target-Specific Changes

**EMBED Assisted Modifications:**
- EMBER-guided editing with intelligent suggestions
- Context-aware modification recommendations
- Automated consistency checking

**Manual Modifications:**
- Direct file editing with user control
- Manual consistency verification
- Traditional editing patterns

## EXECUTION SEQUENCE

### 1. Determine Modification Approach

Based on {{edit_target}} from previous step:

**For Documentation Editing:**
- Load documentation files for editing
- Present modification options (add sections, update content, fix errors)
- Apply changes with EMBER assistance or manual input

**For Configuration Editing:**
- Load workflow configuration files
- Present configuration options (parameters, settings, integrations)
- Apply configuration changes with validation

**For EMBER Integration Editing:**
- Load EMBER integration settings
- Present integration options (prompts, interaction patterns, context rules)
- Apply EMBER integration modifications

**For Project Structure Editing:**
- Load project structure definitions
- Present structure options (scanning rules, analysis patterns, file filters)
- Apply structure modifications

### 2. Execute Mode-Appropriate Modifications

**EMBED Assisted Mode:**
- Use EMBER to analyze current state and suggest modifications
- Present suggestions to user for approval
- Apply approved changes automatically
- Monitor for consistency and coherence

**Manual Mode:**
- Present editing interface for direct modifications
- Allow user to make changes manually
- Provide basic validation and consistency checks
- Track all manual modifications

### 3. Maintain Edit Integrity

- Create backups before modifications
- Track all changes with timestamps
- Maintain version history
- Ensure modifications don't break existing functionality

### 4. Validate Modifications

- Check syntax and format validity
- Verify logical consistency
- Confirm integration compatibility
- Test basic functionality where possible

### 5. Proceed to Review

**EMBED Assisted:** Auto-proceed with confidence scoring
**Manual:** Proceed after user confirmation

## SUCCESS METRICS

- Modifications executed successfully for selected target
- Edit integrity maintained with proper backups
- Changes validated for consistency and functionality
- Correctly proceeded to review step based on mode
