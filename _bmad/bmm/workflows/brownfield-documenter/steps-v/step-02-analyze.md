# Step 2: Brownfield Documenter Validation Analysis

## STEP GOAL:

Execute comprehensive validation analysis using target-specific validation methods and automated testing.

## COMPREHENSIVE ANALYSIS: Target-Specific Validation

**Automated Analysis:**
- Scripted validation execution
- Metric collection and analysis
- Automated test case execution
- Performance benchmarking

**Interactive Analysis:**
- User-guided validation walkthrough
- Manual verification processes
- Expert review and assessment
- Collaborative validation approach

## EXECUTION SEQUENCE

### 1. Determine Analysis Approach

Based on {{validation_target}} from previous step:

**For Documentation Analysis:**
- Load documentation validation scripts
- Execute completeness, accuracy, and consistency checks
- Analyze documentation structure and formatting
- Validate cross-references and links

**For Configuration Analysis:**
- Load configuration validation frameworks
- Check parameter validity and ranges
- Verify integration compatibility
- Test configuration consistency

**For EMBER Integration Analysis:**
- Load EMBER validation test suites
- Execute interaction pattern testing
- Analyze response quality and accuracy
- Validate context preservation

**For Project Analysis Validation:**
- Load project analysis validation tools
- Verify scanning accuracy and completeness
- Check structure recognition quality
- Validate analysis comprehensiveness

**For Complete Workflow Analysis:**
- Execute full workflow validation suite
- Test end-to-end functionality
- Validate integration points
- Perform comprehensive quality assessment

### 2. Execute Validation Tests

**Automated Mode:**
- Run validation test suites
- Collect performance metrics
- Generate detailed analysis reports
- Identify issues and anomalies

**Interactive Mode:**
- Guide user through validation checklists
- Collect user feedback and observations
- Document manual verification results
- Track validation progress

### 3. Analyze Results

Process validation results:
- Calculate validation scores and metrics
- Identify patterns and trends
- Determine severity levels for issues
- Generate improvement recommendations

### 4. Validate Analysis Quality

Ensure analysis completeness:
- Check test coverage adequacy
- Verify analysis methodology correctness
- Confirm result reliability
- Validate analysis conclusions

### 5. Proceed to Reporting

**Automated:** Auto-proceed with analysis results
**Interactive:** Proceed after user review confirmation
