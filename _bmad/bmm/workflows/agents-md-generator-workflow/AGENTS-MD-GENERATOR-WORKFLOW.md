# AGENTS.MD Generator Workflow

**Version:** 6.0.0-alpha.23
**Date:** 2026-01-24
**Workflow Type:** Comprehensive AGENTS.MD Generation and Management

## Overview

This workflow provides a comprehensive framework for generating, managing, and maintaining AGENTS.MD files across projects. It includes all necessary steps, tools, templates, and best practices for creating high-quality, standardized AGENTS.MD documentation.

2. **Steps Directory** - Organized steps for different workflow phases
3. **Templates Directory** - Standardized templates for consistent output
4. **Tools Directory** - Utility scripts and tools for workflow execution
5. **Documentation Directory** - Comprehensive documentation and guides

## Workflow Execution

### Prerequisites

1. **System Requirements:**
   - Operating System: Linux, macOS, or Windows
   - Shell Environment: Bash, Zsh, or compatible
   - Required Tools: Git, Make, Python 3.8+, Node.js 14+
   - Optional Tools: Docker, Kubernetes, Cloud CLI tools

2. **Dependencies:**
   - See `docs/DEPENDENCIES.md` for detailed dependency requirements
   - Install dependencies using `make install-dependencies`

3. **Configuration:**
   - Copy `config.example.yaml` to `config.yaml`
   - Update configuration values for your environment
   - Validate configuration using `make validate-config`

### Quick Start

1. **Clone and Setup:**
   ```bash
   # Clone the workflow repository
   git clone <repository-url>
   cd _bmad/bmm/workflows/agents-md-generator-workflow
   
   # Install dependencies
   make install-dependencies
   
   # Validate configuration
   make validate-config
   ```

2. **Execute Workflow:**
   ```bash
   # Execute the complete workflow
   make execute-workflow
   
   # Execute specific phase
   make execute-phase-1
   make execute-phase-2
   make execute-phase-3
   
   # Execute specific step
   make execute-step-1
   make execute-step-2
   # ... etc
   ```

3. **Monitor and Validate:**
   ```bash
   # Monitor workflow execution
   make monitor-workflow
   
   # Validate workflow results
   make validate-workflow
   
   # Generate workflow reports
   make generate-reports
   ```
