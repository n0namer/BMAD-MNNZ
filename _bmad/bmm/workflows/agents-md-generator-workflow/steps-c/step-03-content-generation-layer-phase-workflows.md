# Step 3: Content Generation — Phase Workflows

## Phase 1: Discovery

### Workflow Steps:
1. Загрузка требований BMAD
2. Анализ пользовательских требований
3. Выявление ограничений и зависимостей
4. Определение scope и целей

### Decision Points:
- Полнота требований
- Валидация scope
- Идентификация ограничений

### Output Artifacts:
- Анализ требований
- Определение scope
- Документация ограничений

---

## Phase 2: Planning

### Workflow Steps:
1. Проектирование 8-слойной архитектуры
2. Мэппинг требований на слои
3. Планирование стратегии генерации контента
4. Определение критериев валидации

### Decision Points:
- Валидация архитектуры
- Мэппинг требований
- Стратегия контента

### Output Artifacts:
- Архитектурный дизайн
- Requirement mapping
- Content plan

---

## Phase 3: Implementation

### Workflow Steps:
1. Генерация layer content
2. Создание task templates
3. Разработка примеров
4. Построение reasoning frameworks

### Decision Points:
- Качество контента
- Эффективность шаблонов
- Актуальность примеров

### Output Artifacts:
- Layer content
- Task templates
- Примеры
- Reasoning frameworks

---

## Phase 4: Validation

### Workflow Steps:
1. Проверка соответствия контента (compliance)
2. Проверка лимитов размера файлов
3. Верификация совместимости с GitHub Copilot
4. Генерация финального вывода

### Decision Points:
- Проверка compliance
- Проверка размера
- Проверка совместимости

### Output Artifacts:
- Final AGENTS.MD
- Final copilot instructions
- Compliance report
